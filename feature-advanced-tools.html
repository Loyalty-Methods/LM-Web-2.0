Programs Grow. Rules Multiply.

What starts as a handful of promotions becomes hundreds of rules. Base earn. Partner earn. Tier multipliers. Promotional bonuses. Each rule interacts with the others. Without structure, you get unknown outcomes. Which rules fired? In what order? Why did the member get 150 points instead of 250? That's a liability problem. An audit problem. A finance problem.

Rule Organization

Predictable Execution. Auditable Outcomes.

Keep programs organized as they grow. Rules are grouped by business function—base earn separate from promotions, partner rules isolated per partner, tier logic in its own area. Different teams can own different sections. When a transaction comes through, execution follows a defined path. Every decision is logged. Finance can trace outcomes back to specific rules. Partner billing calculates accurately because the system knows which rules fired and in what order.

  • Organize rules into logical groupings by business function
  • Different teams manage different areas of the program
  • Gate rules control whether entire sections activate
  • Defined execution order eliminates unknown outputs
  • Logging supports audit and reconciliation

Custom Logic

Handle Edge Cases Without Custom Development

When standard configuration reaches its limits, RCX provides programmable logic. Create custom qualifications that reference any attribute in the system: activity data, member profile, preferences, transaction history. Define calculated fields that can be reused across rules. Validate logic against test data before it goes live. Custom logic becomes immediately available across APIs, UIs, and the data warehouse.

  • Create custom logic for complex qualifications
  • Reference any attribute: activity, member, preferences, history
  • Define reusable calculated fields across rules
  • Validate against test data before deployment
  • Immediately available across APIs and UIs

Built for Teams. Built for Audit.

The tools that keep complex programs manageable over time.

Different teams. Different areas.

Partner teams manage partner rules. Tier teams manage tier logic. Each team sees what they need. Changes stay isolated. Collisions stay rare.

Every decision. Fully traced.

Transaction logs show which rules triggered, which actions executed, and what data informed the decision. Disputes resolve faster because every decision is logged.

Each partner as a mini-program.

Separate funding, billing, and reimbursement logic per partner. Agreements stay isolated. Finance sees clean settlement with full traceability.

Structure that lasts.

Programs that run for years stay organized. New promotions drop into the right folder. Rules don't fall into the void. The system scales with the business.
